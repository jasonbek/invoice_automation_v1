## 1. Seat Selection — New Booking Type

**Problem:** When a supplier sends a separate invoice containing only seat selection charges (no full flight itinerary), the routing agent would misclassify it as a tour because no flight signals were present.

**Solution:** Added `seat_selection` as a first-class booking type throughout the pipeline.

| `app/agents/extractors/seat_selection.py` | **New.** Outputs Seat Screen 1 (Summary) + Seat Screen 2 (Details) — same CBO screens as the seat charge sections appended to a full flight, but used when the seat invoice arrives standalone. `commissionAmount` hardcoded to `"0%"`. Description field includes a per-flight seat assignment list. |
| `app/agents/extractors/__init__.py` | Imported `seat_selection_ext`; registered `"seat_selection"` in `EXTRACTOR_MAP`. |

## 2. File-Based Commission Lookup (Air Canada & WestJet)

**Problem:** Air Canada and WestJet commission rates were hardcoded as simplified Python string constants in `flight.py`. This caused several issues:
- Rules were slightly inaccurate vs. the official docs (e.g., BV/BQ/LGT → 0% was applied to all regions, but the AC doc shows it only applies to Transatlantic).
- Missing categories: Flight Passes (4%), Group Travel (5% North America).
- No way to handle time-limited promotions (e.g., Lufthansa Business class promo).
- Updating rates required a code change and redeploy.

**Solution:** Commission rate tables are now loaded at runtime from `.md` files in `docs/Commissions/`. The hardcoded rate tables were removed from `flight.py`; only business logic (inclusions, exclusions, ACTOT requirement, mixed fare rules, online/interline definitions) remains in the vendor rule constants. The agent reads the real documents and picks the correct rate.

**Adding/updating commission docs going forward:** Drop a `.md` file in `docs/Commissions/` and run `modal deploy app/main.py`. No code changes needed.

### Commission rationale in invoiceRemarks
The agent is now instructed to include one rationale line in `invoiceRemarks` (Screen 1) explaining which rate was selected and why. Examples:
- `"Commission: 4% — AC North America, Economy Y (Appendix 5)"`
- `"Commission: 8% — LH Group promo Mar2026, Business J, plating LH220 (overrides 5% base)"`
- `"Commission: 0% — WestJet Call Centre booking (non-commissionable)"`

## 3. Tourcan Rule Fixes

### 3a. No passenger screen (Screen 3)
**Problem:** Tourcan invoices were generating a Flight Screen 3 (Passengers) section that doesn't exist in ClientBase for Tourcan bookings.

**Solution:** Added `_NO_PASSENGER_SCREEN_VENDORS = {"tourcan"}` in `flight.py`. When the ruleSet is `tourcan`, Screen 3 is omitted from both the schema instructions and the output format. The prompt says "2 sections required" and the extraction instruction adjusts accordingly.

### 3b. TOTAL CREDIT = commission, not a discount
**Problem:** The agent was sometimes treating the negative "TOTAL CREDIT" line on Tourcan invoices as a promotional saving or discount and including it in `invoiceRemarks`.

**Solution:** `TOURCAN_RULES` now explicitly states: *"This negative amount IS the agency commission — it is NOT a discount or promotional saving. Do NOT include it in invoiceRemarks as a saving or discount."*

### Files changed
| File | Change |
|---|---|
| `app/agents/extractors/flight.py` | `TOURCAN_RULES` rewritten. `_NO_PASSENGER_SCREEN_VENDORS` set added. Prompt template made conditional: `{section3_schema}`, `{section_count}`, `{output_format}` variables allow Screen 3 to be omitted per vendor. |
