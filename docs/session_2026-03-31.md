# Session Log — 2026-03-31

## Overview
Four areas addressed: API resilience (529 overload handling), token efficiency (max_tokens right-sizing), form improvements (`.txt` support + attachment validation), and flight commission rationale moved to a dedicated `agentremarks` field.

---

## 1. API Resilience — Retry Hardening

**Problem:** A 529 "Overloaded" error surfaced during a new traveller profile upload. The Anthropic SDK's built-in `max_retries=6` exhausted before the API recovered. Connection drops and timeouts also had no application-level protection.

**Solution:** Added application-level retry loops with exponential backoff on top of the SDK retries, covering `APIStatusError` (529), `APIConnectionError`, and `APITimeoutError`. Backoff schedule: 30s → 60s → 120s → 240s → 300s (capped), up to 8 additional attempts per call.

Also fixed `service_fee.py` which was missing `max_retries=6` on its client instantiation.

### Files changed

| File | Change |
|---|---|
| `app/agents/extractors/base.py` | Added `asyncio` import; replaced fixed-delay 529 retry with exponential backoff loop covering 529, `APIConnectionError`, `APITimeoutError`; increased app retries from 3 → 8. |
| `app/agents/markdown_agent.py` | Same retry pattern added around `client.messages.create`. |
| `app/agents/routing_agent.py` | Same retry pattern added around `client.messages.create`. |
| `app/agents/extractors/service_fee.py` | Added `max_retries=6` to client; same retry pattern added. |

### Retry behaviour summary
| Layer | Handles | Max additional wait |
|---|---|---|
| SDK (`max_retries=6`) | 429 rate limits, minor transients | ~60s |
| App retries (8 attempts) | 529 overload, connection drops, timeouts | ~40 min total |

## 2. Token Efficiency — max_tokens Right-Sizing

**Problem:** `tour.py` and `cruise.py` both had `max_tokens=4096` despite producing only 2 sections each. Typical output is 400–800 tokens.

**Solution:** Reduced max_tokens to match realistic output size for each extractor.

### Files changed

| File | Change |
|---|---|
| `app/agents/extractors/tour.py` | `max_tokens` reduced from 4096 → 2000. |
| `app/agents/extractors/cruise.py` | `max_tokens` reduced from 4096 → 1500. Note: if a cruise with a very long port-by-port itinerary is truncated, bump back to 2000. |

---

## 3. Form Improvements — `.txt` Support + Attachment Validation

**Problem 1:** `.txt` files were not accepted by the upload form, and `.txt` files inside zip archives were silently dropped.

**Problem 2:** The submit button was active even with no file attached, allowing empty submissions.

### Files changed

| File | Change |
|---|---|
| `app/main.py` (form HTML) | Added `.txt` to `accept` attribute and hint text on the file input. |
| `app/main.py` (form JS) | Submit button disabled on page load; re-enabled only when at least one file is selected. Secondary guard in submit handler prevents submission if file count is zero. |
| `app/main.py` (`_expand_upload`) | Added `.txt` → `text/plain` content-type mapping so `.txt` files inside zips are extracted instead of silently skipped. |

---

## 4. Flight — Commission Rationale Moved to `agentremarks`

**Problem:** The commission rationale line (e.g. `"Commission: 4% — AC North America, Economy Y (Appendix 5)"`) was being written into `invoiceRemarks`, which maps to the client-visible invoice remarks field in ClientBase. It should be internal/agent-only.

**Solution:** Added `agentremarks` as a new field in the Flight Screen 1 JSON output, positioned after `invoiceRemarks`. The commission rationale now goes there. `invoiceRemarks` reverts to seat selections only.

### Files changed

| File | Change |
|---|---|
| `app/agents/extractors/flight.py` | `_SECTION1_SUMMARY_ONLY` (AC/WestJet/ADX): `invoiceRemarks` description updated to seats only; `agentremarks` added with commission rationale description. `_SECTION1_FULL` (Tourcan/Expedia/generic): `agentremarks` added as `""`. `_COMMISSION_SECTION_TEMPLATE`: rationale instruction updated from `invoiceRemarks` → `agentremarks`. |

### Field layout — Flight Screen 1 (AC/WestJet/ADX)
```
invoiceRemarks  → seat selections block (client-visible)
agentremarks    → commission rationale line (agent-only)
```

> Note: The UI.Vision macro also needs updating to map `agentremarks` to the Agent Remarks field in the CB flight screen.
